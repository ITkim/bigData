Get방식과 Post방식에 대해.

두 방식 모두, '서버에 요청을 하는 메소드'이다.
클라이언트가 서버에 요청을 할 때, 제공해야 하는 자원이 있다고 치자. 예를 들면, 어떤 홈페이지의 로그인 페이지에서 로그인을 하는 경우이다. 아이디와 패스워드는 클라이언트가 작성한 후, 그 정보를 서버에 요청하여 클라이언트가 작성한 아이디와 패스워드가 올바른 것인지 검사를 해야한다. 위의 예시를 보듯, 요청에는 자원을 보내야 하는 경우가 존재한다.


1. Get에 데이터는 url뒤에 ?뒤 오고, key와 value쌍으로 넣어야한다. 물음표 뒤의 표기를 기준으로 &는 key와 value쌍을 구분하는 구분자이다.

예시를 보자면,
www.example.com?id=mommoo&pass=1234



@Get과 Post는 무슨 차이인가?
-> 결과 상의 차이는, Get은 url에서 input값이 표기되지만
-> Post방식을 사용하면 url에 input값이 표기되지 않는다는 것.

작동 상의 차이는?
-> Get은 '가져오는 것'이고 Post는 '수행하는 것' 이다.

Get은 서버에서 어떤 데이터를 가져와서 보여주지만, 서버의 값이나 상태는 바꾸지 않는다. ex)게시판의 리스트, 글 보기 기능.

반면 Post는 서버의 값이나 상태를 바꾸기 위해 사용된다. 예를 들어 글쓰기를 하면 글의 내용이 DB에 저장이 되고 수정을 하면 DB값이 수정된다.

Get방식은 큰 데이터를 보낼 수 없고, Post방식은 가능하다.
